# 第一阶段实际开工清单

## 目标

第一阶段的目标不是做完整产品，而是做出一个可用的 **P0 MVP 主链**：

1. 原生应用能打开
2. 权限流程成立
3. 能完成系统声音 + 麦克风录音
4. 能落盘生成音频文件
5. 能自动跑通 Whisper
6. 能自动跑通 transcript 解析
7. 能自动跑通 Gemma 纪要生成
8. 能在结果页看到 transcript 与纪要

---

## P0 范围

### P0-1 工程骨架

- 建立 macOS 原生应用工程
- 使用 `SwiftUI`
- 建立基础目录结构
- 建立应用状态模型

交付标准：

- 工程可编译
- 应用可启动
- 有基础的三页导航骨架

### P0-2 权限流程

- 实现麦克风权限检查
- 实现系统录制相关权限检查
- 权限未开时给出引导
- 权限就绪后允许进入录音页

交付标准：

- 首次启动能明确告诉用户缺什么权限
- 已授权后能进入录音页

### P0-3 录音主链

- 录系统声音
- 录麦克风声音
- 默认双开
- 可开始录音
- 可停止录音
- 生成单一可供 Whisper 使用的音频文件

交付标准：

- 一次完整录音能稳定落盘
- 输出文件可被 `ffmpeg`/Whisper 正常识别
- 第一阶段录音实现目标系统版本为 `macOS 15+`

### P0-4 Session 存储

- 每次处理生成独立 session 目录
- 保存：
  - 原始音频
  - `transcript.json`
  - `transcript.txt`
  - `meeting_minutes.md`
  - 日志文件

交付标准：

- 每次运行都不会覆盖上一次结果
- 文件结构清晰可追溯

### P0-5 Whisper Runner

- 固定调用已确认命令模板
- 安全构造参数数组
- 执行转写
- 输出到 session 目录

交付标准：

- 输入音频后能自动产出 `transcript.json`
- 命令失败时有清晰错误信息

### P0-6 Transcript Parser

- 优先解析 `segments`
- 次优解析 `words`
- 兜底纯文本
- 输出统一 transcript 文本格式

交付标准：

- 能生成稳定的 `transcript.txt`
- 时间戳格式统一

### P0-7 Gemma Runner

- 固定调用已确认命令模板
- 基于 transcript 构造中文纪要 prompt
- 输出 Markdown 纪要

交付标准：

- 能自动产出 `meeting_minutes.md`
- 输出结构符合既定模板

### P0-8 结果页

- 展示 transcript
- 展示会议纪要
- 支持复制 transcript
- 支持复制会议纪要
- 支持导出 Markdown

交付标准：

- 用户在应用中可直接消费结果
- 不需要回到终端看文件

### P0-9 处理状态与错误提示

- 展示处理阶段：
  - 保存音频
  - 正在转写
  - 整理 transcript
  - 生成纪要
- 失败时展示可理解错误信息

交付标准：

- 用户知道当前卡在哪一步
- 不直接暴露底层 CLI 噪音

---

## 第一阶段建议实施顺序

1. 工程骨架
2. 权限页
3. 录音页假流程
4. 双路录音与落盘
5. Session 存储
6. Whisper Runner
7. Transcript Parser
8. Gemma Runner
9. 结果页
10. 端到端联调

---

## 第一阶段先不做

以下内容明确不进入第一阶段：

### 1. 实时转写

原因：

- 会把录音与 AI 处理强耦合
- 复杂度高
- 不是 MVP 核心成功条件

### 2. 实时纪要生成

原因：

- 没必要
- 成功率和性能风险都更高

### 3. 历史记录系统

原因：

- 当前 session 文件夹已足够满足调试与追溯
- UI 级历史管理不是第一阶段关键路径

### 4. 设置页

原因：

- 当前参数已锁定
- 不需要过早开放模型和命令配置

### 5. 多模型切换

原因：

- 当前环境基线已确认
- 先保证一条链路稳定

### 6. 录音设备高级配置

原因：

- 第一版只要系统声音 + 麦克风默认可用
- 不先做复杂输入源选择器

### 7. Web 版 / CLI 版 / API 服务版

原因：

- 已锁定产品形态为 macOS 原生应用
- 先把主产品做成

### 8. 云同步与账号体系

原因：

- 明显超出 MVP 范围

### 9. 自动标题优化、智能标签、会议分类

原因：

- 属于体验增强，不是主链阻塞项

### 10. BlackHole 兼容模式

原因：

- 先尝试原生系统音频采集
- 只有原生方案不稳定时才进入下一阶段

---

## P0 完成定义

当以下条件全部满足时，第一阶段可以结束：

1. 应用可启动
2. 权限流程可走通
3. 能完成一次双路录音
4. 能自动生成原始音频
5. 能自动生成 `transcript.json`
6. 能自动生成 `transcript.txt`
7. 能自动生成 `meeting_minutes.md`
8. 应用内可查看 transcript 与纪要
9. 用户不需要手工执行命令

---

## 第二阶段候选项

仅在 P0 稳定后再讨论：

1. BlackHole 兼容模式
2. 历史记录页
3. 重新生成不同风格纪要
4. 会议信息编辑
5. 导出更多格式
6. 桌面通知
7. 自动命名 session
